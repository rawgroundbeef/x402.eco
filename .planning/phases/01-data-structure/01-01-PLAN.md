  <name>Task 1: Create TypeScript types for ecosystem entries</name>
  <files>lib/types/ecosystem.ts</files>
  <action>
Create a new TypeScript file with types for ecosystem entries.

Define the following types:

1. `CategorySlug` - Union type of the 4 category slugs:
   - "client-integrations"
   - "services-endpoints"
   - "infrastructure-tooling"
   - "learning-community"

2. `EcosystemEntry` - Interface for a single entry:
   - name: string (required)
   - description: string (required)
   - url: string (required)
   - category: CategorySlug (required)
   - logo: string (optional - path relative to public/, e.g., "/logos/entry-name.png")
   - tags: string[] (optional - for future filtering)

3. `CategoryMeta` - Interface for category display info:
   - slug: CategorySlug
   - title: string (display name like "Client-Side Integrations")
   - description: string (brief category description)

4. Export a `CATEGORIES` constant array of CategoryMeta for all 4 categories with proper titles:
   - client-integrations -> "Client-Side Integrations"
   - services-endpoints -> "Services & Endpoints"
   - infrastructure-tooling -> "Infrastructure & Tooling"
   - learning-community -> "Learning & Community"

Do NOT create a JSON schema file - TypeScript types are sufficient for this static site.
  </action>
  <verify>
Run `pnpm exec tsc --noEmit` to verify types compile without errors.
Check that lib/types/ecosystem.ts exists and exports the required types.
  </verify>
  <done>
TypeScript types EcosystemEntry, CategorySlug, CategoryMeta, and CATEGORIES are defined and compile successfully.
  </done>
